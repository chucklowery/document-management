# Output and Security Acceptance Specifications

**Project:** Organizational Knowledge and Work System

# Feature AS-027 — Governed Output and Delivery

**Stories:** US-070, US-071  
**Requirements:** REQ-IO-004 through REQ-IO-009

## Scenario AS-027.1 — Material output has reproducibility manifest

**Given** a report is generated from exact Revisions and a Projection Definition,  
**When** the report is finalized,  
**Then** an Output Manifest records sources, definition, parameters, policies, tool version, generated time, completeness, limitations, authorization scope, and integrity information.

## Scenario AS-027.2 — Delivery retry does not duplicate intended delivery

**Given** a Subscription delivery times out after the destination accepted it,  
**When** the system retries using the same delivery idempotency key,  
**Then** the destination receives no second intended delivery unless the Output Contract permits it,  
**And** each Delivery Attempt remains recorded.

## Scenario AS-027.3 — Correction governs prior output

**Given** a delivered report relied on a source record later corrected,  
**When** the correction is accepted,  
**Then** the applicable output policy regenerates, supersedes, withdraws, expires, or flags the report,  
**And** records the selected action and affected Output Manifest.

# Feature AS-028 — Derived-Store Revocation

**Stories:** US-072  
**Requirements:** REQ-SC-001 through REQ-SC-006; REQ-IO-008

## Scenario AS-028.1 — Revocation reaches every derived representation

**Given** a user's access to a sensitive Resource is revoked,  
**And** the Resource appears in a text index, vector index, cache, facet, saved result, and pending delivery,  
**When** the revocation is effective,  
**Then** each affected representation is removed, restricted, invalidated, or cryptographically erased within policy,  
**And** unresolved propagation is visible to authorized stewards,  
**And** verification evidence identifies each checked store.

## Scenario AS-028.2 — Policy service failure does not weaken access

**Given** the system cannot evaluate a required authorization policy,  
**When** a restricted retrieval or delivery is attempted,  
**Then** the system fails closed for the affected disclosure,  
**And** enters or reports the applicable degraded state.
