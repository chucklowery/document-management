# Findability and Information Experience Context Extension

**Project:** Organizational Knowledge and Work System

The Information Experience and Findability context owns navigation composition, wayfinding, labels, aliases, facets, refinement behavior, result previews, Saved Workspaces, personalization presentation, and findability evaluation.

Domain contexts retain authoritative meaning and lifecycle. Information Access and Retrieval owns Query execution and ranking. Identity, Access, and Governance owns permission policy. System Operations and Delivery owns configuration execution and recovery.

The context consumes stable identities, Revisions, Relationships, Trails, authority, freshness, and permitted retrieval results. It publishes Vocabulary Revisions, Navigation Model Revisions, Refinement State, Saved Workspace Revisions, and findability evaluation evidence.

A shared label or navigation view does not create a shared runtime domain model.
