# Acceptance Specifications

# Feature AS-037 — Partial Outcomes and Resume

**Stories:** US-093, US-094  
**Requirements:** REQ-LO-005, REQ-LO-006, REQ-LO-007, REQ-LO-008

## Scenario AS-037.1 — Mixed outcome remains itemized

**Given** a partial-acceptance operation contains several items,  
**When** the operation stops with mixed results,  
**Then** the summary identifies the mixed outcome,  
**And** every consequential item identifies its result and next action.

## Scenario AS-037.2 — Resume preserves the boundary

**Given** work stops after a durable checkpoint,  
**When** an operator resumes with compatible definition Revisions,  
**Then** work continues from the declared boundary,  
**And** completed effects are not repeated,  
**And** pending work remains accounted for.

# Feature AS-038 — Governed Overload

**Stories:** US-095  
**Requirements:** REQ-LO-009, REQ-LO-010

## Scenario AS-038.1 — Backpressure is visible and governed

**Given** incoming work exceeds safe processing capacity,  
**When** Backpressure State becomes active,  
**Then** an authorized operator can see backlog, oldest work age, rates, capacity, and effects on freshness,  
**And** each delayed, prioritized, throttled, expired, or rejected item follows the identified Admission Policy,  
**And** acknowledged work is not silently lost.

# Feature AS-039 — Alignment Review

**Stories:** US-096  
**Requirements:** REQ-LO-011, REQ-LO-012

## Scenario AS-039.1 — Differences remain reviewable

**Given** an alignment review examines named boundaries,  
**When** a material difference is found and an approved restoration runs,  
**Then** the review records its definition, boundaries, time, result, and unresolved differences,  
**And** the restoration records its plan Revision, checkpoints, item outcomes, verification result, and unresolved items.

# Feature AS-040 — Correlated Operational Records

**Stories:** US-097  
**Requirements:** REQ-LO-013

## Scenario AS-040.1 — Delivery records connect to intake

**Given** an accepted input produces a managed change, derived view, Output Manifest, and Delivery Attempt,  
**When** an authorized steward follows stable correlation references,  
**Then** each stage is navigable according to policy,  
**And** each record retains its identity, authority, and applicable time.
