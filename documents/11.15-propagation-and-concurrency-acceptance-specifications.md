# Propagation and Concurrency Acceptance Specifications

**Project:** Organizational Knowledge and Work System

# Feature AS-025 — Replay, Backfill, and Propagation Health

**Stories:** US-067, US-068  
**Requirements:** REQ-SM-008 through REQ-SM-011; REQ-OP-001 through REQ-OP-003

## Scenario AS-025.1 — Replay is idempotent and bounded

**Given** a known event boundary must be replayed,  
**When** an authorized operator initiates replay,  
**Then** the system records the boundary, processing definition, actor, reason, checkpoints, and outcome,  
**And** previously applied events do not create duplicate domain effects.

## Scenario AS-025.2 — Propagation dashboard explains lag

**Given** an accepted source Revision,  
**And** the backlink index is current but the retrieval index and one subscription are delayed,  
**When** an Operations Steward inspects propagation health,  
**Then** the system separately identifies each derived representation, checkpoint, lag, state, and failure,  
**And** identifies whether lag values are measured or estimated.

## Scenario AS-025.3 — Read-your-writes disclosure

**Given** a workflow requires read-your-writes consistency,  
**When** a change is acknowledged before the retrieval index is updated,  
**Then** a subsequent read resolves the acknowledged Revision through an authorized path or explicitly states that retrieval propagation is pending.

# Feature AS-026 — Concurrent Contribution Reconciliation

**Stories:** US-069  
**Requirements:** REQ-SM-012 through REQ-SM-014

## Scenario AS-026.1 — Incompatible changes are not overwritten

**Given** a human and an Automated Agent prepare changes from the same base Revision,  
**And** the human change is accepted first,  
**When** the agent submits its incompatible change with the earlier Expected Base Revision,  
**Then** the system detects the mismatch,  
**And** preserves both acknowledged contributions,  
**And** does not overwrite the human Revision.

## Scenario AS-026.2 — Reconciliation is explicit

**Given** two conflicting contributions are preserved,  
**When** a Reconciliation Authority merges selected parts,  
**Then** the system records both base and contribution Revisions, authority, policy, rationale, and resulting Revision,  
**And** historical contributions remain addressable.
