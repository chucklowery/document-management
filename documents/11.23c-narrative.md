# Feature AS-041 — Literate Operation Conformance

**Stories:** US-098, US-099  
**Requirements:** REQ-LO-015, REQ-LO-016, REQ-LO-017

## Scenario AS-041.1 — Operation is explained in human order

**Given** a consequential operation has contracts, states, checks, and recovery procedures,  
**When** its Operational Narrative is reviewed,  
**Then** it explains purpose, authority, inputs, outputs, invariants, state transitions, failures, observability, governance, recovery, and examples,  
**And** every consequential reference identifies an exact Revision or durable selection rule.

## Scenario AS-041.2 — Stale executable reference fails conformance

**Given** an Operational Narrative references a check that no longer exists,  
**When** documentation validation runs,  
**Then** the Narrative Conformance Result identifies the stale reference and evaluated boundary,  
**And** conformance does not pass.
