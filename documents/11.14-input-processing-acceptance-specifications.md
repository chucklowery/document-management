# Input Processing Acceptance Specifications

**Project:** Organizational Knowledge and Work System

# Feature AS-024 — Idempotent Input and Visible Failure

**Stories:** US-065, US-066  
**Requirements:** REQ-SM-001 through REQ-SM-007; REQ-IO-001 through REQ-IO-003

## Scenario AS-024.1 — Duplicate event creates one domain effect

**Given** an Input Contract defines an idempotency scope,  
**And** the same external change is delivered twice with the same idempotency key,  
**When** both deliveries are processed,  
**Then** only one intended domain effect is created,  
**And** duplicate handling is recorded.

## Scenario AS-024.2 — Invalid input is quarantined

**Given** incoming information fails semantic validation,  
**When** it cannot be safely accepted,  
**Then** a Quarantine Record identifies source, contract, findings, time, retry eligibility, and responsible steward,  
**And** no accepted Resource Revision is created from the invalid payload,  
**And** the input is not searchable or deliverable as accepted information.

## Scenario AS-024.3 — Late arrival preserves domain time

**Given** a value observed on Monday arrives after a value recorded on Wednesday,  
**When** the late value is accepted on Thursday,  
**Then** its Monday observation time and Thursday recorded time are preserved,  
**And** affected current Projections are recalculated without rewriting prior records.
