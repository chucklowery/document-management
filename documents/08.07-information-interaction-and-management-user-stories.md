# Information Interaction and Management User Stories

**Project:** Organizational Knowledge and Work System

## US-084 — Continue Across Channels
**As a user,** I want a notification, export, or delivered output to reopen the relevant identity, filters, time boundary, and checkpoint, so that channel changes preserve meaning.

## US-085 — Review Daily Change
**As an information seeker,** I want changes since a known checkpoint categorized, so that I can review additions, revisions, withdrawals, relevance changes, failures, and unresolved items.

## US-086 — Navigate Accessibly
**As a user of assistive or non-pointer interaction,** I want all discovery and management paths operable with equivalent status and error information, so that I can complete the same task.

## US-087 — Govern Vocabulary Change
**As a vocabulary steward,** I want to preview affected navigation, queries, mappings, outputs, and users before changing a term, so that semantic impact is visible.

## US-088 — Preview a Configuration Change
**As a configuration steward,** I want a dry run and impact preview before a consequential change, so that I can inspect scope and recovery behavior.

## US-089 — Execute a Bounded Bulk Action
**As a configuration steward,** I want bulk operations to show selected scope and per-item outcomes, so that partial success and failure are recoverable.

## US-090 — Evaluate Findability With Real Tasks
**As a findability evaluator,** I want representative task evidence across audiences and access contexts, so that design changes are judged by user outcomes rather than one metric.
