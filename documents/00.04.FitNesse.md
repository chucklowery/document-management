# FitNesse Vision Document

## Vision Statement

FitNesse will be the leading open-source collaborative specification and acceptance testing platform that enables business stakeholders, testers, and developers to define, validate, and continuously verify software behavior through executable specifications.

By combining documentation, collaboration, and automated verification in a single platform, FitNesse will help organizations build the right software with confidence while maintaining living documentation that evolves alongside their systems.

---

# Product Purpose

FitNesse exists to bridge the communication gap between business and technology teams by transforming requirements into executable acceptance tests.

The platform enables teams to:

* Define requirements collaboratively
* Express expected behavior using examples
* Execute those examples against real systems
* Detect regressions quickly
* Maintain documentation that reflects actual system behavior

FitNesse serves as both a collaboration tool and a test automation platform, creating a shared source of truth for software behavior.

FitNesse remains a powerful acceptance testing framework with an integrated wiki and executable specification model. It is widely respected for enabling Specification by Example and Acceptance Test Driven Development (ATDD).

However, modern software teams increasingly expect:

* CI/CD integration

FitNesse will evolve into a modern executable specification platform that:

* Makes business requirements executable

* Supports cloud-native deployments

* Enables AI-assisted specification creation and maintenance

# Product Principles

## 1. Specification First

Requirements should be expressed before implementation.

Specifications become executable assets rather than static documents.

---

## 2. Collaboration Over Handoffs

All stakeholders should contribute to understanding system behavior.

The platform should encourage shared ownership of requirements.

---

## 3. Documentation Must Be Verifiable

Documentation without verification becomes stale.

Every significant specification should be executable.

---

## 4. Simplicity Matters

Business stakeholders should be able to contribute without programming expertise.

The barrier to participation should remain low.

---

## 5. Open and Extensible

FitNesse should remain:

* Open source
* Language agnostic
* Framework agnostic
* Integration friendly

This aligns with FitNesse's long-standing commitment to openness and adaptability.

# Product Differentiators

## Executable Specifications

Unlike traditional documentation tools, FitNesse specifications can be executed directly against applications.

## Integrated Wiki

Unlike many test automation frameworks, FitNesse provides built-in collaborative documentation.

## Business Readability

Tests are designed to be understood by non-developers.

## Technology Independence

Supports multiple languages and implementation technologies through fixtures and adapters.
