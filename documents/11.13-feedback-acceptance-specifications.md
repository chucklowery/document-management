# Feedback Acceptance Specifications

**Project:** Organizational Knowledge and Work System

# Feature AS-023 — Saved Search, Change Digest, and Feedback

**Stories:** US-062, US-063, US-064  
**Requirements:** REQ-IR-009 through REQ-IR-014; REQ-SC-004

## Scenario AS-023.1 — Change Digest compares from acknowledged checkpoint

**Given** a user saved a search and acknowledged its prior results,  
**And** qualifying Resources were added, revised, superseded, and withdrawn,  
**When** the user requests changes since the checkpoint,  
**Then** the Change Digest identifies each permitted change category according to the comparison policy,  
**And** preserves the checkpoint and Saved Search Revision used.

## Scenario AS-023.2 — Feedback remains contextual

**Given** a user marks a result irrelevant for a Query and task,  
**When** the judgment is recorded,  
**Then** the system records the Query context, result identity, assessor, judgment, corpus, Ranking Policy, and time,  
**And** does not represent the judgment as proof that the source is false or unimportant.

## Scenario AS-023.3 — Ranking regression blocks adoption

**Given** a proposed Ranking Policy Revision,  
**And** a governed Evaluation Set,  
**When** evaluation shows an unapproved material loss in recall or safety,  
**Then** adoption is blocked unless an authorized time-bounded exception records scope, rationale, monitoring, and rollback.
